Signing a CSR

What is a CSR?

A Certificate Signing Request (CSR) contains the public key and subject information. The private key remains with the requester, providing security as it never leaves their system.

When to use CSR signing:

  • Key pair generated on target server
  • Private key managed by another team
  • Compliance requires key separation
  • HSM-protected private keys

Subject Alternative Names

Leave empty to use SANs from the CSR, or override by entering custom SANs. All hostnames the certificate should cover must be listed.

The signed certificate will be marked as "External" since the private key is not managed by this system.
