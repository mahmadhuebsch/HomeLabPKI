About Certificates

What are End-Entity Certificates?

End-entity certificates (also called leaf certificates) are issued to servers, clients, or users. They cannot sign other certificates and are used for authentication, encryption, and digital signatures.

Actions:

  • New Certificate - Generate a new key pair and certificate
  • Sign CSR - Sign an external Certificate Signing Request
  • Import - Import an existing certificate for tracking

Certificate Types:

  • TLS Server - For HTTPS web servers
  • TLS Client - For client authentication
  • Code Signing - For signing software
  • Email (S/MIME) - For email encryption

Best Practices:

  • Use short validity periods (1 year max)
  • Include all necessary SANs
  • Use appropriate key usage extensions
  • Monitor expiration dates
