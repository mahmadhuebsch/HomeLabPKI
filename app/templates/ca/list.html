About Root CAs

What is a Root CA?

A Root Certificate Authority is the trust anchor of a PKI hierarchy. It is self-signed and used to sign Intermediate CAs or directly issue certificates.

Actions:

  • New Root CA - Create a new self-signed Root CA
  • Import - Import an existing Root CA for tracking

Best Practices:

  • Keep Root CA keys offline and secure
  • Use long validity periods (10+ years)
  • Use strong key algorithms (RSA 4096 or ECDSA P-384)
  • Create Intermediate CAs for day-to-day operations
  • Limit direct certificate signing from Root CA
